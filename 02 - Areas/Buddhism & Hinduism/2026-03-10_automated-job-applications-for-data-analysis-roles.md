# Automated job applications for data analysis roles

> **Conversation Overview**

This conversation centered on job search assistance for Pilli Shreyas, a final-year Computer Science student at MLR Institute of Technology in Hyderabad. Shreyas is seeking Data Analyst and AI/ML roles in Hyderabad and came with a resume already in hand. The conversation began with Shreyas asking Claude to autonomously apply for jobs on his behalf through LinkedIn and other platforms, which Claude explained was not possible due to lack of account access, browser automation capabilities, and platform terms of service. Claude redirected toward legitimate automation tools (LazyApply, Simplify.jobs, Jobscan) and offered to assist with job search preparation instead.

Claude completed several concrete deliverables throughout the conversation: a curated list of job search links across LinkedIn, Naukri, Indeed, Glassdoor, and Foundit tailored to Shreyas's Python, SQL, LangChain, and ML skill set; a LinkedIn headline trimmed to 43 characters reading "Data Analyst | Python · ML · AI Agent Dev"; a human-sounding cover letter initially written at approximately 400 words, then trimmed to approximately 200 words at Shreyas's explicit request, and finally provided as plain copyable text for manual use. Claude also summarized Shreyas's top three projects (NeoAttend Agent, Open Source Agent, IMDB Data Analysis) with impact framing and tools used, and highlighted his top three achievements (1st place at AI Tech CODEVERSE, 1st runner-up at Zignasa 2k24 MERN hackathon, Oracle Cloud Infrastructure AI Foundations certification). The session concluded with Claude generating a formatted Excel job tracker containing 20 job listings with direct apply links, skill match notes, a status column, and platform details.

Shreyas demonstrated a preference for concise, copy-paste-ready outputs and direct deliverables over explanations. He corrected the cover letter length by specifying "trim it down to exactly ~200 words" after Claude sought clarification. His resume highlights skills in Python, SQL, Pandas, LangChain, Google Gemini, TensorFlow, Scikit-Learn, REST APIs, Flask, and AWS, with the Oracle Cloud Infrastructure AI Foundations certification adding credibility beyond academic work.

**Tool Knowledge**

Claude used Python with the `openpyxl` library to generate the Excel job tracker, installed via `pip install openpyxl --break-system-packages -q` in the execution environment. The working pattern for styled Excel output involved layering `PatternFill`, `Font`, `Border`, and `Alignment` objects per cell, with merged cells for the title row, frozen panes set at `A3`, and explicit column width assignments using `get_column_letter`. Hyperlink-style formatting was achieved by setting font color to `0563C1` with `underline="single"` on the link column rather than using Excel's native hyperlink object, which avoided compatibility issues. The cover letter `.docx` was generated using the `docx` npm package via Node.js, with `Packer.toBuffer()` writing to the output path. File outputs were saved to `/mnt/user-data/outputs/` for both the Excel and Word files.
